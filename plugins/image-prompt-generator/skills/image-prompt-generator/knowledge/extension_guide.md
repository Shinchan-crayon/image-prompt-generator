# 知识库扩展规范

## 何时读取

只有在新增或修改分类、意图、情绪、隐喻、案例或模型适配时读取。正常生成提示词时不
加载本文件。

新增知识必须可被路由、可被比较，并能通过自检确认结构完整。

## 新增文章分类

每个分类必须包含：

```text
分类名称
分类说明
推荐视觉方向
推荐构图
推荐风格
推荐色彩
常见错误
```

分类说明中应写清与最相近分类的边界；只有边界复杂时再增加独立说明。优先补充现有分类，
不要为一个品牌或单篇新闻创建分类。

## 新增意图或情绪

必须写出可观察的视觉变化，例如镜头距离、光线强弱、动作节奏和色彩对比。禁止只列
“高级、震撼、未来感”等形容词。

## 新增视觉隐喻

每条隐喻记录：

```text
适用关系：变化 / 约束 / 竞争 / 信任 / 增长等
现实场景：
主体：
动作：
为什么成立：
不适用场景：
撞图风险：
```

若隐喻不能用现实场景拍摄，优先改造成实物演示，而不是退回悬浮概念图。

## 新增案例

案例编号持续递增，并包含：

```text
文章摘要：
模式与目标模型：
文章理解：
视觉策略：
最终 Prompt：
```

新增案例必须覆盖旧案例没有覆盖的分类、意图、情绪、构图或模型组合。案例用于校准，
不能被复制成固定答案。

新增案例建议同时记录以下检索元数据，便于后续检索和覆盖检查：

```text
主分类：
写作意图：
主情绪：
关系类型：
构图类型：
```

## 新增模型

只在确认模型的语言偏好或参数能力后更新 `knowledge/model_language.md`。无法确认时，
使用通用自然语言 Prompt，不编造参数。

## 质量检查

每次扩展后：

1. 运行 `python3 scripts/check_skill.py`；
2. 使用一个既有主题和一个新增主题检查输出；
3. 检查新增规则是否让 SKILL.md 变长，能否改为条件加载；
4. 检查是否仍然只支持封面和正文配图两种运行模式。
