# 文章理解与视觉判断

## 何时读取

在选择分类、情绪或画面之前读取。本文只负责把输入整理成可验证的视觉决策，不负责写最终 Prompt。

需要细化时再定点读取：

- 分类不明确：`article_classification.md`
- 作者写作目的不明确：`intent_library.md`
- 情绪强度不明确：`emotion_library.md`
- 需要落到画面：`visual_strategy.md`

## 快速判断

先分开回答三个问题：

1. **发生了什么**：输入明确陈述了哪些事实？
2. **作者如何解释**：作者把事实连接成了什么关系、变化或判断？
3. **读者应带走什么**：读者看完后应相信、理解或重新认识什么？

文章理解不是摘要。摘要追求覆盖，视觉判断只保留一个最值得被看见的关系。

## 证据标记

所有判断都使用以下标记：

- `[FACT]`：输入直接陈述、可定位的事实。只代表“文章这样写”，不代表已做外部核验。
- `[INFERENCE]`：由标题、措辞、因果结构或篇幅重点推导出的判断，必须附依据。
- `[ASSUMPTION]`：为了继续设计而暂时采用的假设，必须说明需要确认什么。
- `[UNKNOWN]`：输入不足，不能可靠判断。不得用常识或想象补齐。

不要把作者的预测、企业宣传、标题判断或引用者观点标成 `[FACT]`。它们作为文本事实可以写成“文章声称……”，其真实性仍未核验。

## 七项输出

严格按以下顺序形成内部分析卡：

```markdown
## 主题
[FACT/INFERENCE] 一句话说明文章正在讨论的主要问题。

## 分类
主分类：
辅助分类：无 / 一个
依据：

## 作者意图
意图：
[INFERENCE] 作者希望读者从什么认知或行动转向什么认知或行动。
依据：

## 核心观点
[INFERENCE] 一句话，只包含一个关系、变化或因果。

## 情绪
主情绪：
强度：低 / 中 / 高
依据：

## 证据
- [FACT] 输入中的关键事实或原句含义：
- [FACT] 支撑关系的第二条证据：
- [UNKNOWN/ASSUMPTION] 尚未确认的信息：

## 视觉策略
模式：封面模式 / 正文配图模式
通过【一个主体】在【一个地点】发生【一个动作或状态变化】，表达【核心观点】。
优先载体：现实摄影 / 桌面实物演示
```

最终对用户输出时遵循对应模板；除非模板要求，不展示完整分析卡和隐藏推理。

## 字段判断规则

### 主题

主题写“文章在解决什么问题”，不写关键词串。

- 合格：`企业采用 AI 时，竞争重点正从模型分数转向工作流可靠性。`
- 不合格：`AI、企业、模型、Agent、效率。`

### 分类

- 选择对核心观点解释力最大的一个主分类。
- 只有第二分类会明显改变视觉主体或机制时，才增加一个辅助分类。
- 名词出现不等于分类成立。融资新闻中的 GPU 可能只是公司业务背景。
- 两类冲突时，优先选择文章篇幅最多、结论最依赖的那一类。

### 作者意图

作者意图描述“希望读者发生什么变化”，而不是文章形式。

- `介绍新产品`过于宽泛。
- `让读者相信持续可用性比单次性能更重要`可以指导画面。
- 标题与正文冲突时，以正文反复论证和结论段为主，并标记为 `[INFERENCE]`。

### 核心观点

核心观点必须通过四项测试：

1. 去掉品牌、人物与事件名后仍成立。
2. 只表达一个主要关系。
3. 包含方向，例如增加、转移、受限、替代、依赖或分化。
4. 能被一个主体和一个动作表现。

若句中出现“以及、同时、并且”，通常需要继续收敛。

### 情绪

- 情绪来自作者的语气和读者应有的感受，不来自题材刻板印象。
- 技术新闻不默认“未来”，融资不默认“财富”，安全文章不默认“危机”。
- 选择一个主情绪并标注强度；辅助情绪只有在它改变光线或动作时才记录。
- 情绪不得推翻事实关系。例如“突破”不能把有限改进画成无限能力。

### 证据

优先寻找：

1. 标题与导语中的判断；
2. 反复出现的因果或对比；
3. 数据、案例、产品变化等事实；
4. 结尾给出的读者结论；
5. 被强调的限制、例外与不确定性。

每个核心观点至少需要一条直接证据和一条结构性依据。证据不足时使用 `[UNKNOWN]`，不要用更强烈的视觉替代缺失信息。

### 视觉策略

视觉策略不是候选创意列表。它必须：

- 指定两种运行模式之一；
- 选择一个主体、地点、动作和焦点；
- 优先现实摄影；抽象关系优先桌面实物演示；
- 保留事实边界，不把趋势画成既成事实；
- 能在三秒内看懂主要关系。

## 冲突处理

| 冲突 | 处理 |
|---|---|
| 标题夸张，正文克制 | 采用正文证据，降低情绪强度 |
| 公司宣传与独立事实混合 | 分别标记，宣传结论写为 `[INFERENCE]` 或“文章声称” |
| 同时包含多个行业 | 按核心观点选择主分类，不按名词数量投票 |
| 作者立场不清 | 标记 `[UNKNOWN]`，采用中性解释策略 |
| 输入只有标题 | 可判断暂定主题与模式，其余标记 `[ASSUMPTION]` |
| 视觉上需要补充细节 | 只补充不改变事实的场景细节，如桌面、手部和自然光 |

## 最小示例

输入：某旧款设备获得新的本地 AI 功能，文章认为硬件竞争正在转向长期软件服务。

```markdown
## 主题
[FACT] 旧设备通过软件更新获得新能力。

## 分类
主分类：产品测评
辅助分类：商业
依据：文章关注使用价值延长及竞争方式变化。

## 作者意图
意图：观点
[INFERENCE] 让读者重新认识产品差异化的来源。

## 核心观点
[INFERENCE] 持续软件服务正在延长硬件的实际价值。

## 情绪
主情绪：成长
强度：中
依据：重点是能力延续，而非颠覆或危机。

## 证据
- [FACT] 旧款设备获得新功能。
- [FACT] 文章把更新与长期使用价值联系起来。
- [UNKNOWN] 更新覆盖范围与持续时间未提供。

## 视觉策略
模式：封面模式
通过【一位用户】在【自然光书桌前】继续操作【有使用痕迹的旧设备】，表达【软件服务延长硬件价值】。
优先载体：现实摄影
```

## 常见失败

- 把文章摘要当成核心观点，导致画面承载五六个信息点。
- 把“读者可能感兴趣”当成作者意图。
- 没有证据仍推断公司动机、市场结果或技术能力。
- 分类、意图和情绪各自生成一个画面，最后拼成多焦点画面。
- 为了视觉冲击放大结论，破坏文章原有的限制条件。
