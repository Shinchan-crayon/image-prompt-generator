# 构图决策库

## 何时读取

已经确定核心观点、主体、地点和动作，需要决定镜头、空间关系、留白与焦点时读取。
本文件负责组织一个画面，不负责增加主体或补充第二个观点。

## 快速决策

1. 先确定画面唯一焦点：手部动作、人物判断、物体状态或关键界面四选一。
2. 再按任务选择镜头：人物关系用中景，精细判断用近景，资源与流程用俯视，规模才用广角。
3. 封面先划定标题安全区，再安排主体；正文先保证关系可读，再保留必要呼吸空间。
4. 辅助元素只用于解释主动作，不得靠高亮、尺度或色彩形成第二焦点。

## 单一焦点

- 主体只能有一个；多人协作时，共同工件是主体，人物是动作执行者。
- 焦点只使用一到两种手段建立，例如尺度加清晰度，或光线加局部色彩。
- 高对比边缘、脸、手、关键屏幕和运动方向不能同时争夺注意力。
- 无法在三秒内说出“正在看什么、发生什么”，说明构图仍需简化。

## 场景构图库

| 构图类型 | 适用场景 | 推荐镜头 | 留白 | 常见错误 |
|---|---|---|---|---|
| 单人判断 | 审核、选择、接管、学习、医疗确认 | 肩后中近景或操作近景，人物视线与手部指向同一对象 | 封面留在人物视线前方；正文只保留低纹理边缘 | 脸、手、屏幕各成焦点；人物摆拍看镜头 |
| 共同工件 | 协作、交接、评审、人机分工 | 中景或轻微俯拍，让所有动作汇聚到同一文件、设备或样本 | 留在动作外侧，不切断参与者与工件关系 | 握手、围桌合影、每人一块无关屏幕 |
| 桌面机制 | 资源分配、规则变化、抽象流程、成本转移 | 垂直或轻微斜俯视，手与物件保持真实比例 | 封面可在桌面一侧保留连续材质；正文四周均匀呼吸 | 物件过多、像棋盘或信息图、依赖密集标签 |
| 真实 UI 操作 | 软件交互、任务执行、权限核对、产品体验 | 肩后视角或屏幕与手同框的近景 | 留在墙面、桌面或低信息界面区域，不遮挡关键操作 | 正面大截图、虚假密集文字、多个漂浮设备 |
| 现场设备 | GPU、制造、医疗、自动驾驶、工业安全 | 人物与设备关系用中景；局部机制用近景 | 用通道、墙面、地面或景深背景形成自然留白 | 设备英雄照、无限走廊、忽略操作者与尺度 |
| 单路径过程 | 工作流、审批、物流、训练步骤 | 俯视或轻微轴测，路径从一个清晰入口走向出口 | 沿路径外侧留白，不在路径中间放标题 | 路径回环、箭头过多、超过四个阶段 |
| 同主体对照 | 前后变化、方案差异、能力边界 | 固定视角、尺度、光线和背景，只改变一个变量 | 两侧保持相近负空间；封面标题只占一侧非关键区域 | 情绪化左右分屏、同时改变人物和环境 |
| 共享入口 | 平台、商业竞争、渠道、规则治理 | 正面中景、俯视分拨台或单点汇聚构图 | 入口旁保留低干扰区域，避免连接线穿过留白 | Logo 星系、中心发光球、无意义辐射结构 |
| 瓶颈队列 | 算力、供应、审批、服务容量 | 轻微俯拍或侧向中景，让等待与狭窄节点同框 | 留在队列外侧；标题不得遮住瓶颈位置 | 用警告图标代替瓶颈、队列无限延伸 |
| 规模现场 | 产能、部署范围、基础设施扩张 | 有前景参照物的广角中远景 | 天空、墙面或整洁通道可作留白 | 只追求宏大、主体过小、重复元素失去含义 |
| 规则边界 | 政策、合规、安全区、运营范围 | 平视中景或俯视检查台，让规则与被检查对象同框 | 沿边界外侧留白，不让标题跨越规则线 | 法槌、印章、抽象红线、把出台等同执行 |

## 封面构图

### 适用原则

- 使用单场景非对称构图，主体通常占画面约 35%–55%。
- 动作方向应朝向画面内部或标题安全区的相反方向。
- 标题安全区先于装饰元素确定；留白是可排版背景，不是空出来的杂乱区域。
- 广角只用于“规模本身”是观点的文章，默认优先中景或中近景。

### 常用结构

| 结构 | 适用场景 | 镜头 | 标题留白 | 常见错误 |
|---|---|---|---|---|
| 左文右图 | 人物操作、产品使用、工作现场 | 主体位于右侧三分之一，中景或近景 | 左侧约 35%–45%，低纹理连续背景 | 动作朝画外；亮屏或手伸入标题区 |
| 右文左图 | 设备现场、人物观察、道路场景 | 主体位于左侧三分之一，视线向内 | 右侧约 30%–40%，避免窗框和设备边缘 | 主体与标题争夺最高对比 |
| 上文下图 | 竖版人物、桌面俯拍、教育场景 | 主体压低但不贴底 | 上方约 25%–35%，背景连续 | 吊灯、窗框或头部切割标题区 |
| 下文上图 | 设备、手部动作、道路远景 | 主体抬高，保留稳定地面或桌面 | 底部约 20%–30%，低细节 | 零散文具、道路标线或手部进入留白 |
| 中心偏置 | 无标题或短标题、强单物体关系 | 焦点偏离中心，保留运动方向 | 只在明确平台无需大标题时使用 | 假居中、四周平均散落辅助元素 |

## 正文配图构图

### 适用原则

- 解释关系优先于标题安全区，概念节点不超过四个。
- 关系用距离、接触、顺序、容器或状态表达，连接线只是最后手段。
- 流程保持单向可追踪；对照保持同一主体、视角、尺度和环境。
- 正文可以使用真实 UI、桌面演示、流程或对照，但它们都是正文配图模式内的构图子策略。

### 镜头选择

- **操作近景**：适合权限确认、医疗核对、学习反馈、设备细节；必须保留动作上下文。
- **肩后视角**：适合真实 UI 与人的判断关系；避免屏幕占满画面。
- **中景**：适合人物、设备和环境三者关系，是大多数现实场景的默认选择。
- **俯视**：适合资源、步骤、桌面机制和共同工件；避免物件均匀铺满。
- **轻微轴测**：只用于真实空间难以看清的层级或流程，保持可拍摄材质和合理尺度。
- **广角中远景**：只用于部署规模、产线或道路范围，必须有前景人物或物体提供尺度。

## 留白检查

每次明确四项：

1. **位置**：左、右、上、下或场景自然空区。
2. **比例**：封面给出大致百分比；正文说明需要何种呼吸空间。
3. **材质**：墙面、桌面、天空、地面、景深背景或低信息界面。
4. **禁入元素**：主体、脸、手、关键状态、高亮边缘、连接线或强纹理。

合格示例：

`画面左侧约 40% 保留连续浅灰墙面和柔和阴影，不放人物、手部、亮屏或高对比设备边缘，用于后期标题。`

## 构图失败时

- **多焦点**：保留最能证明核心观点的动作，其余元素降级为背景或删除。
- **留白不可用**：移动主体和动作，不用模糊或压暗关键内容伪造安全区。
- **解释依赖文字**：改用位置、数量、顺序、材质或状态差异。
- **像海报**：降低戏剧对比，缩小事件规模，回到可观察的现场动作。
- **像信息图**：减少节点，替换抽象图标为真实物件或真实 UI。
- **像广告**：加入使用痕迹、责任动作和现实环境，移除品牌中心与完美表面。
