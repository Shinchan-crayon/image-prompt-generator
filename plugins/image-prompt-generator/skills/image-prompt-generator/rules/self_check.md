# 自检与重设计

生成后先执行硬门槛，再执行评分。完整评分规则见 `rules/quality_scoring.md`。

## 硬门槛复核

任一失败都不得输出：

1. **观点**：表达的是文章观点，而不是新闻事件或关键词吗？
2. **单一性**：是否只有一个核心观点、一个主要主体和一个视觉焦点？
3. **易懂性**：目标读者能否在三秒内理解主要关系？
4. **场景**：是否存在一个明确动作或状态变化？
5. **媒体感**：封面是否像专业科技媒体，正文图是否像清晰编辑插图？
6. **留白**：封面是否有位置明确、真正可用的标题安全区？
7. **独特性**：是否依赖机器人、Logo、代码雨、蓝色霓虹等默认 AI 图像套路？
8. **模型适配**：Prompt 语言是否符合目标模型？是否避免编造模型参数？
9. **负面约束**：Negative Prompt 是否针对当前画面的主要失败风险？
10. **场景化**：是否明确了现实地点、主体、动作与光线？
11. **长度**：中文是否为 50–60 个汉字，英文是否为 35–45 个单词？

## 评分复核

硬门槛全部通过后，对文章理解、观点提炼、视觉策略、场景或必要隐喻、构图留白、
风格光线色彩和 Prompt 可执行性进行 10 分评分。现实操作场景已经能直接表达观点时，
不得为了评分强行增加象征隐喻。

- `9.0–10.0`：允许输出。
- `<9.0`：返回核心观点、视觉策略或构图步骤重新设计。
- 禁止虚增分数，禁止只追加“高级、电影感、极简、8K”等形容词后再次输出。

## 输出自检

使用简短清单：

```text
- 表达观点而非新闻：通过/未通过
- 单一焦点：通过/未通过
- 容易理解：通过/未通过
- 符合科技媒体或解释型配图目标：通过/未通过
- 避免常见撞图套路：通过/未通过
- 场景、动作和光线明确：通过/未通过
- Prompt 长度符合目标：通过/未通过
- 综合评分：X/10
```

只输出最终通过的方案。综合评分必须不低于 9 分。若重新设计过，可写“已重设计”，
不要展示失败草稿、逐步推理或虚构的精确依据。
