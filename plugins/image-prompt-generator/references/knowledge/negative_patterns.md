# 视觉错误案例库

## 何时读取

视觉策略已经形成，需要检查同质化、陈词滥调和错误替代物时读取。本文件用于更换失败的
场景或生成针对性 Negative Prompt，不负责提出第二套创意。

本文件解释常见失败为什么无效，以及如何修复。

| 错误模式 | 为什么不好 | 替代方式 |
|---|---|---|
| 机器人站着摆姿势 | 用“机器人”代替 AI 观点，没有因果关系 | 让真实人物使用自动化工具完成一个动作 |
| Logo 居中 | 只说明新闻主体，不表达文章意义 | 去掉品牌后提炼长期变化或用户影响 |
| 蓝色光污染 | 把科技感简化成颜色，破坏真实材质 | 使用自然光与一个受控技术状态 |
| 廉价赛博朋克 | 场景与多数商业、教育内容无关 | 使用办公室、商店、家庭或工业现场 |
| 代码雨 | 不解释模型、产品或工作流机制 | 使用真实界面、任务卡或流程中的动作 |
| 默认 AI 脸 | 容易怪异、同质化且与观点无关 | 只在人的体验是核心时使用自然人物 |
| 漂浮 HUD | 增加信息噪音且缺少空间逻辑 | 把信息落到真实屏幕、纸张或设备上 |
| 粒子特效 | 不能表达关系，只制造“生成感” | 用光线、材质和主体动作建立焦点 |
| 六边形背景 | 高频科技模板，没有内容含义 | 选择与行业真实相关的结构和环境 |
| 握手式协作 | 陈词滥调，无法说明如何协作 | 展示人做判断、系统执行重复步骤 |
| 灯泡表示创新 | 过度通用，无法区分文章 | 展示旧流程被具体新动作替代 |
| 火箭表示增长 | 把商业关系变成空洞成功符号 | 使用订单、产线、资源或空间状态变化 |
| 棋盘表示竞争 | 高频且容易把复杂竞争过度简化 | 使用货架、接口、供应链或真实选择场景 |
| 锁和盾牌表示安全 | 只贴标签，不说明风险对象 | 展示访问边界、隔离动作或验证流程 |
| 左右分屏对比 | 容易出现两个焦点和海报化 | 在同一主体上表现前后状态变化 |
| 多设备漂浮 | 像广告素材拼贴，缺少主关系 | 选择一个设备和一个关键使用动作 |
| 数据图表乱填 | 容易编造数字和趋势 | 使用无具体数值的真实业务状态或已知数据 |
| 电影海报化 | 情绪压过解释，正文配图尤其失真 | 降低对比和特效，保留清晰空间关系 |
| 纯白电商产品图 | 只有物体，没有文章观点 | 加入真实使用环境与人的动作 |
| “高级感”黑金模板 | 风格先于内容，所有主题长得一样 | 从材质、光线和构图建立克制品质 |

## 使用方法

1. 找出当前方案最可能落入的两到四个错误模式。
2. 将它们写入针对性的 Negative Prompt。
3. 若错误来自核心隐喻，不要只写负面词，直接更换场景。
