# 模型与语言适配

## 何时读取

每次任务在目标模型已知或需要选择 Prompt 语言时读取。本文件只决定语言、长度与已确认的
模型写法，不改变核心观点、场景或构图。

## 基本规则

目标模型决定最终 Prompt 的语言，不改变中文分析与中文输出标题。

| 目标模型 | Prompt 语言 | 写法 |
|---|---|---|
| GPT Image | 中文或中英混合 | 50–60 个汉字，优先真实摄影与生活化场景 |
| Midjourney | 英文 | 35–45 个单词，具体场景、动作、光线和色调 |
| Flux | 英文 | 35–45 个单词，使用清晰英文描述现实空间关系 |
| Seedream | 中文或中英混合 | 50–60 个汉字，优先中文真实摄影描述 |
| Ideogram | 英文 | 35–45 个单词；需要画面文字时才写文字内容 |
| ThinkAI Image 2 `gpt-image-2-4k` | 中文或中英混合 | 50–60 个汉字，按 GPT Image 的场景化真实摄影写法 |
| ThinkAI Nano `nano-banana-2` | 中文或中英混合 | 50–60 个汉字，直接描述主体、动作、光线与比例 |
| 火山引擎方舟 / Seedream | 中文或中英混合 | 50–60 个汉字，按 Seedream 的中文场景化写法 |
| OpenAI GPT Image | 中文或中英混合 | 50–60 个汉字，优先具体场景和自然语言 |
| Google Nano Banana | 中文或中英混合 | 50–60 个汉字，优先清晰主体、动作、光线和构图 |

若用户要求通过本 Skill 的 ThinkAI Image 2 或 ThinkAI Nano 连接器生图，目标模型分别
固定按 `gpt-image-2-4k` 或 `nano-banana-2` 处理。若此前为 Midjourney、Flux 或 Ideogram
生成了英文 Prompt，进入 ThinkAI 审核前必须重新适配并再次展示，不得把未审核的新版本
直接送去生成。

若用户显式选择火山引擎，按 Seedream 处理；选择 OpenAI 时按 GPT Image 处理；选择
Google 时按 Nano Banana 处理。实际渠道模型与当前 Prompt 目标模型不一致时，必须重新
适配、展示和审核；配置渠道不能自动批准 Prompt。

## 未指定模型

默认按 GPT Image 处理，使用中文或中英混合 Prompt，并在输出中标注：

`目标模型：未指定，按 GPT Image 通用写法处理。`

## 未知模型

若用户指定的模型不在表中：

1. 不猜测该模型的专用参数。
2. 使用结构清晰的中文 Prompt；若用户明确要求英文，则使用英文。
3. 标注使用“通用自然语言 Prompt”。

长度按输出语言回退：中文 50–60 个汉字，英文 35–45 个单词。

## 模型参数

- 只使用已知且用户明确需要的参数。
- 比例优先写进自然语言描述。
- Midjourney 可在用户明确指定该模型时附加常见比例参数，如 `--ar 16:9`。
- 不确定版本、风格参数或负面参数时，不输出对应命令。
