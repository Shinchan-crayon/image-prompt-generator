# 文章级多图工作流

## 适用边界

当用户提供完整文章，并明确要求整篇文章配图、自动规划多张图片、生成文章视觉包或继续一个
已有文章任务时读取本文件。

文章工作流是封面模式和正文配图模式之上的编排层，不是第三种视觉模式。单张封面或单张
正文配图继续使用 `SKILL.md` 的单图工作流。

## 总流程

严格按以下顺序执行：

```text
保存完整文章
→ 自动制定图片规划
→ 展示规划并等待确认
→ 逐张生成和展示 Prompt
→ 逐张批准或修改
→ 全部保留 Prompt 均已批准
→ 受控并发批量生图
→ 全部成功后创建交付包
```

任何步骤都不得跳过。文章规划确认不等于 Prompt 批准，Prompt 批准也不能覆盖后续修改。

## 创建任务

图片渠道需要在创建文章任务前完成配置。通过文件或标准输入传递文章，禁止把完整文章塞入
命令参数。

```bash
python3 scripts/article_workflow.py init \
  --article-file '<ARTICLE_FILE>' \
  --title '<ARTICLE_TITLE>' \
  --provider thinkai-image2
```

脚本返回任务 ID 和 `runs/<任务ID>/` 路径。后续操作只使用该任务，不创建重复任务。

## 自动图片规划

先对全文建立文章理解卡和结构摘要，再选择真正需要图片帮助的段落。不得按标题数量机械
配图，也不得为了凑数量选择重复观点。

默认建议：

- 一张封面，表达全文唯一核心观点。
- 三至六张正文配图，分别解释机制、过程、对比、数据关系或关键转折。
- 短文可以少于三张正文图；只有图片确实增加理解时才保留。
- 同一观点只安排一张图。

每个规划项目必须包含：

- 稳定 ID，只能使用小写字母、数字和连字符。
- `cover` 或 `content` 模式。
- 对应标题和原文范围。
- 图片作用。
- 比例、渠道尺寸和画面风格。

按插件根目录的 `templates/image_plan.md` 展示规划。用户修改时重新保存完整规划；用户明确确认后才能
进入 Prompt 阶段。

替换完整规划会重建项目列表，并使此前所有 Prompt 与批准状态失效。旧 Prompt 文件即使
仍留在任务目录，也不得继续使用；必须按新规划重新生成和审核。

## 逐图 Prompt

封面项目调用封面模式规则，正文项目逐项调用正文配图模式规则。每个正文项目只读取自己的
标题、所属段落和必要上下文，不用全文观点覆盖段落解释任务。

每张图仍需经过文章理解、视觉策略、长度控制和质量门。通过后将完整输出保存为：

```json
{
  "image_goal": "图片作用",
  "core_viewpoint": "唯一观点",
  "visual_metaphor": "视觉隐喻与理由",
  "composition": "主体、动作、镜头、背景、留白和焦点",
  "style": "画面风格",
  "prompt": "最终正向 Prompt",
  "negative_prompt": "需要避免的元素"
}
```

使用 `set-prompt` 保存后，向用户完整展示当前图片方案并停止。用户明确批准后，使用
`approve-prompt` 记录批准。任何字段变化都会增加版本并使该图旧批准失效。

## 审核进度

每次批准或修改后按插件根目录的 `templates/batch_review.md` 汇报：

- 已批准数量。
- 等待审核数量。
- 需要修改数量。
- 已删除数量。
- 当前是否达到全量批准门槛。

用户可以明确删除正文图片项目；封面不能删除。删除项目不再参与全量批准和批量生成。

## 断点恢复

用户要求继续任务时，先运行：

```bash
python3 scripts/article_workflow.py status --run '<RUN_ID>'
```

根据真实状态继续：

- `plan_review`：继续规划审核。
- `prompt_review`：找到第一个未批准项目继续。
- `ready`：全部 Prompt 已批准，可以进入生成。
- `generating`：跳过已成功图片，从下一张继续。
- `blocked`：先处理失败或结果不确定项目。
- `complete`：不重复生成，直接检查或重建交付包。

结果不确定但本地已有完整产物时，按 `references/rules/batch_generation.md` 的 `resolve-item` 规则
核验并接纳；不能验证时继续保持阻塞。

不得依赖对话记忆猜测进度，也不得直接手工修改 `workflow-state.json`。
