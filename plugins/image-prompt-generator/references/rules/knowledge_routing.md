# 知识路由规则

知识库按任务逐层加载，不要每次读取全部文件。

## 启动固定加载

开始分析时读取：

1. `references/knowledge/article_understanding.md`
2. `references/knowledge/model_language.md`
3. `references/knowledge/scene_photography.md`
4. `references/rules/core_rules.md`
5. `references/rules/quality_scoring.md`
6. `references/rules/negative_rules.md`

## 阶段固定加载

每次任务都会读取，但只在对应阶段加载：

- 设计镜头与留白时读取 `references/knowledge/composition.md`；
- 写最终 Prompt 时读取 `templates/prompt_templates.md`；
- 生成后读取 `references/rules/self_check.md`。

## 条件加载

完成文章理解卡后再选择：

| 判断结果 | 加载文件 |
|---|---|
| 文章分类 | `references/knowledge/article_classification.md` 对应条目 |
| 写作意图 | `references/knowledge/intent_library.md` 对应条目 |
| 主情绪 | `references/knowledge/emotion_library.md` 对应条目 |
| 视觉策略 | `references/knowledge/visual_strategy.md` |
| 需要隐喻 | `references/knowledge/visual_metaphors.md` |
| 需要解释机制 | `references/knowledge/explanatory_visuals.md` |
| 需要标题排版 | `references/knowledge/typography.md` |
| 需要确定色彩 | `references/knowledge/color_language.md` |
| 需要风格校准 | `references/knowledge/editorial_style.md` |
| 需要检查陈词滥调 | `references/knowledge/negative_patterns.md` |

封面模式继续读取 `references/rules/cover_rules.md` 和 `templates/cover_output.md`。
正文配图模式继续读取 `references/rules/content_rules.md` 和
`templates/content_output.md`。

`templates/prompt_templates.md`、`references/knowledge/composition.md` 和
`references/rules/self_check.md`
属于阶段固定资源，不因文章分类变化而跳过。

## 决策优先级

发生冲突时按以下顺序裁决：

1. 文章事实与作者核心观点；
2. 图片只表达一个核心信息；
3. 封面或正文配图的功能目标；
4. 写作意图；
5. 文章情绪；
6. 风格、色彩和品牌参考。

例如，安全事故文章即使属于“产品发布”，也不能使用庆祝式暖光；教程文章即使
主题是 AI，也不能为了科技感牺牲解释清晰度。

## 分类主体路径

分类只提供默认主体路径，不直接决定最终画面。先按下表选择默认路径，再检查核心观点是否
满足覆盖条件。**核心观点可以覆盖分类默认值**，但必须说明切换原因；不得因为分类名称中
出现 AI、芯片或政策就机械套用固定主体。

三条路径的含义：

- `人物行动优先`：人的选择、体验、责任或协作承担主要叙事。
- `实物/过程优先`：设备、材料、空间、队列、边界或物体状态承担主要叙事，人物和手只在
  操作关系不可缺少时出现。
- `界面/机制优先`：低噪音真实界面、实验状态或机制结构承担主要叙事，必要时再加入操作人。

| 分类 | 默认主体路径 | 改走其他路径的触发条件 |
|---|---|---|
| AI 模型 | 界面/机制优先 | 核心是用户完成真实任务或专业人员承担判断责任时，改走人物行动优先 |
| AI Agent | 界面/机制优先 | 核心是委派、确认、接管或团队协作时，改走人物行动优先 |
| 机器人 | 实物/过程优先 | 核心是人机协作、安全责任或用户体验时，改走人物行动优先 |
| 芯片 | 实物/过程优先 | 核心是工程取舍、质检责任或供应链人工决策时，可改走人物行动优先 |
| GPU | 实物/过程优先 | 核心是运维调度、采购取舍或用户等待体验时，可改走人物行动优先 |
| 创业 | 人物行动优先 | 核心是产品结构、资金分配或流程瓶颈时，可改走实物/过程优先 |
| 融资 | 实物/过程优先 | 核心是创始人的关键取舍或投资人的判断责任时，可改走人物行动优先 |
| 商业 | 实物/过程优先 | 核心是顾客体验、销售判断或服务交付时，可改走人物行动优先 |
| 安全 | 实物/过程优先 | 核心是审核、接管、误操作或责任归属时，可改走人物行动优先 |
| 教程 | 界面/机制优先 | 教学重点是实体操作或学习者判断时，分别改走实物/过程或人物行动优先 |
| 工作流 | 实物/过程优先 | 核心是角色交接、审批责任或人的等待体验时，可改走人物行动优先 |
| 赚钱 | 人物行动优先 | 核心是投入产出结构、交易路径或成本转移时，可改走实物/过程优先 |
| 产品测评 | 人物行动优先 | 核心是纯硬件结构、耐久测试或界面机制时，可改走实物/过程或界面/机制优先 |
| 行业分析 | 实物/过程优先 | 核心是从业者共同处境或关键决策者行为时，可改走人物行动优先 |
| 公司分析 | 实物/过程优先 | 核心是管理层取舍、组织协作或员工执行时，可改走人物行动优先 |
| 政策 | 实物/过程优先 | 核心是执法、审核、申请或责任主体的具体行动时，可改走人物行动优先 |
| 自动驾驶 | 人物行动优先 | 核心是传感器、道路边界或车辆决策机制时，可改走实物/过程优先 |
| 医疗 AI | 人物行动优先 | 核心是设备流程、排队机制或匿名化数据结构时，可改走实物/过程优先 |
| 教育 AI | 人物行动优先 | 核心是反馈机制、课程结构或学习路径时，可改走界面/机制优先 |
| 设计 AI | 人物行动优先 | 核心是生成机制、版本关系或素材流程时，可改走界面/机制优先 |
| 办公 AI | 人物行动优先 | 核心是任务流、权限结构或自动化状态时，可改走界面/机制优先 |
| 未来趋势 | 实物/过程优先 | 核心是人们已经开始形成的新行为时，可改走人物行动优先 |

执行顺序：

1. 用主分类取得默认主体路径。
2. 检查核心观点中的主语：是人的判断、物体过程，还是界面机制。
3. 若与默认路径不同，按表中覆盖条件切换并记录一句理由。
4. 再选择具体载体；不得把三条路径同时放进一张图。

## 组合规则

最终视觉策略只保留：

- 一个分类方向；
- 一个主要写作意图；
- 一个主情绪，可附一个弱辅助情绪；
- 一个主要视觉策略；
- 一个核心场景或隐喻。

不得把每个知识库选中的元素全部塞进画面。知识库用于做选择，不是用于堆叠。

## 未命中处理

找不到精确条目时：

1. 选择结构最接近的上位分类；
2. 明确标记为 `[INFERENCE]`；
3. 仍按现实场景、单一动作、明确光线设计；
4. 任务结束后可按 `references/knowledge/extension_guide.md` 增加新条目。
