# 案例 001–025

## CASE-001

- 文章摘要：新模型在长文推理中更稳定，但速度与成本没有同步改善。
- 模式与目标模型：封面模式｜GPT Image｜16:9
- 文章理解：能力进步来自稳定完成复杂任务，而不是所有指标同时跃升。
- 视觉策略：研究员在测试桌前让一叠长任务卡通过单一检查轨道，右侧留标题区。
- 最终 Prompt：`真实摄影，研究员在明亮测试桌前逐张核对长任务卡，卡片稳定通过窄轨，晨光柔和，中性色调，画面克制，右侧留出干净标题空间。`

## CASE-002

- 文章摘要：章节解释上下文窗口扩大后，关键信息仍可能被中间内容淹没。
- 模式与目标模型：正文配图模式｜Seedream｜4:3
- 文章理解：容量增加不等于检索可靠，核心问题是重要信息能否被准确找到。
- 视觉策略：编辑从很长的纸卷中抽出一张关键便签，用单一动作解释检索。
- 最终 Prompt：`真实摄影，编辑在木桌上展开长纸卷，从密集资料中抽出一张关键便签，午后窗光照亮手部，暖灰色调，材质真实，背景简洁清楚。`
