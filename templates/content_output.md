# 正文配图模式输出模板

严格使用以下结构：

```markdown
# 图片目标

一句话说明这张图帮助读者理解当前章节中的什么。

---

# 核心观点

一句话表达当前章节需要解释的一个关系、过程或变化。

---

# 视觉隐喻

说明采用的现实场景、实物演示或解释结构，以及它为何比装饰性画面更清楚。直接场景足够
时，不强制使用象征隐喻。

---

# 构图

- 主体：
- 动作：
- 镜头：
- 背景：
- 留白：
- 焦点：

---

# 风格

说明清晰度、信息密度、真实材质、色彩与可解释光线。

---

# Prompt

目标模型：

最终 Prompt：

长度：中文 50–60 个汉字；英文 35–45 个单词。

优先真实生活场景或桌面实物演示，只保留一个解释关系。

---

# Negative Prompt

针对当前方案列出需要避免的元素。

---

# 自检

- 解释本节而非整篇文章：
- 单一焦点：
- 容易理解：
- 符合解释型配图目标：
- 避免海报化与常见撞图套路：
- 场景、动作和光线明确：
- Prompt 长度符合目标：
- 综合评分：X/10
```
