# 场景化真实摄影

## 何时读取

每次任务在把核心观点转化为可拍摄场景时读取。本文件负责建立人物或实物、地点、动作、
光线和色调，不决定文章分类或最终构图。

## 核心原则

不要告诉模型“画一个概念”，要告诉模型“现实中什么主体在什么地方发生什么变化”。

按分类主体路径选择一种结构：

```text
人物行动：真实摄影 + 具体人物 + 真实地点 + 一个判断或操作动作 + 结果状态 + 光线 + 色调
实物/过程：真实摄影 + 具体实物或设备 + 真实地点 + 位置、数量、队列或状态变化 + 光线 + 色调
界面/机制：真实设备 + 低噪音界面或实验状态 + 一个关键反馈 + 现场光线 + 克制背景
```

人物不是所有科技题材的默认主体。芯片、GPU、政策、流程和行业结构等内容，默认可由
实物、设备、边界或过程承担叙事；只有人的选择、体验、责任或协作不可省略时才加入人物。

## 场景转换

将抽象概念转换为可拍摄场景：

- AI 自动客服：店主在收银台喝茶，手机正在自动回复顾客。
- 工作流连续性：开发者把按小时分隔的任务卡重新排进一周项目板。
- 资源预算：有限筹码随着不同任务在桌面项目板上移动。
- 软件服务延长产品价值：用户继续使用旧设备，新功能在屏幕上自然运行。
- 企业效率：员工完成主要工作，自动系统在旁边处理重复步骤。

## 抽象机制

当正文内容难以直接生活化时：

1. 优先使用桌面摆拍、纸卡、筹码、容器、轨道等真实物件。
2. 优先让物体自身的状态变化解释机制，例如移动、排队、分流、占满、断开或重新组合。
3. 只有操作来源、责任或尺度关系必须可见时才加入一双手或人物；其他情况不强制出现人物或手。
4. 不退回抽象信息图、悬浮图标或随机界面。

## 光线与色调

- 生活化产品、商业服务、教育内容：自然窗光、午后阳光、柔和暖色调。
- 冷静分析、工业技术：中性自然光，可保留少量暖色重点。
- 风险、危机、安全议题：根据语义采用冷静或低饱和色调，不强制温暖气氛。

暖色调是默认策略，不是覆盖文章情绪的固定滤镜。

## 长度压缩

正向 Prompt 只保留对出图有直接影响的信息：

1. 真实摄影；
2. 场景；
3. 主体；
4. 动作；
5. 光线；
6. 色调；
7. 封面必要留白。

把解释原因、设计分析和禁止项放在其他输出区，不塞进正向 Prompt。
