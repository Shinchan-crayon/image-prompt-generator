# 视觉策略决策

## 何时读取

完成主题、分类、作者意图、核心观点、情绪和证据判断后读取。本文负责从分析结果中选择一个可执行画面，不负责生成图片，也不增加运行模式。

## 模式边界

本 Skill **只支持两种模式**：

1. **封面模式**：完整文章的核心观点需要一张编辑主视觉。
2. **正文配图模式**：标题、段落或章节中的关系、过程、结构或变化需要一张解释图。

不得新增“新闻模式、海报模式、社交媒体模式、批量模式、信息图模式”或任何第三种功能。新闻、教程、评测等是文章分类或意图，不是运行模式。

## 三条主体路径

先从 `rules/knowledge_routing.md` 取得分类默认路径，再用核心观点检查是否需要覆盖。三条路径
互相平行，不是“人物失败后逐级降级”的顺序。

### 人物行动优先

- 适用：人的选择、体验、责任、协作或接管本身就是观点。
- 载体：现实人物与真实场景摄影，动作比表情重要。
- 约束：人物必须改变、确认或承担某个结果，不能只是站在设备旁边提供“科技感”。

### 实物/过程优先

- 适用：制造、算力、资源分配、供应、规则边界、流程或物理状态是观点核心。
- 载体：真实物体现场、设备局部、桌面实物演示或同一工件的状态变化。
- 约束：物体的位置、数量、队列、接触、边界或状态变化可以独立承担动作，不强制加入人物。

### 界面/机制优先

- 适用：软件交互、Agent 运行状态、模型测试、权限或反馈机制是观点核心。
- 载体：低噪音真实 UI、设备屏幕与可信实验状态，或可被现实物件解释的机制。
- 约束：人物或手只在操作、确认、责任边界不可省略时出现；界面不能变成随机仪表盘。

不要因为题材是 AI 或科技就跳到抽象插画、机器人、代码雨、HUD、发光大脑或未来城市。

## 单画面契约

每次策略必须填满以下一句话：

`通过【一个主体】在【一个地点】发生【一个动作或状态变化】，表达【一个核心观点】。`

并满足：

- 一个主要主体；
- 一个主要动作；
- 一个视觉焦点；
- 一个现实空间；
- 一种主情绪；
- 一条可见关系。

无法填入时，返回核心观点步骤继续收敛，不建立候选画面拼贴。

## 封面模式

### 目标

在三秒内传达文章“意味着什么”，形成可用于标题排版的科技媒体编辑主视觉。

### 策略选择

- 核心是**变化**：让同一主体完成一次状态转变。
- 核心是**取舍**：让人物把有限资源移向一个明确方向。
- 核心是**竞争**：让多个力量争用同一个焦点，画面仍只保留一个主关系。
- 核心是**限制**：让瓶颈、门槛或边界在现实动作中可见。
- 核心是**信任**：让人物执行检查、确认或接管动作。
- 核心是**趋势**：用现实中的早期信号和未完成路径暗示方向。

### 构图

- 优先非对称单场景，中景或近景。
- 主体通常占画面约 35%–55%。
- 标题安全区必须说明位置、比例和低干扰要求。
- 留白不放关键动作、亮点、高对比纹理或发光线条。
- 视觉隐喻只保留一个，且能用一句话解释与观点的对应关系。

### 风格

- 真实摄影、编辑纪实或可信产品摄影。
- 通过材质、动作和光线建立品质，不用“高级感、未来感、电影感”代替设计。
- 情绪强度服从证据；宣传性文章也不自动采用广告式光效。

## 正文配图模式

### 目标

降低当前章节的理解成本，让读者看懂一个关系、过程、结构或状态变化。

### 策略选择

- 核心是**关系**：两个对象必须由一个明确动作、接触或资源流连接。
- 核心是**过程**：使用一条单向路径，通常三到四个状态，焦点落在关键变化。
- 核心是**结构**：使用真实容器、层级、剖面或模块关系，避免装饰性连接线。
- 核心是**对比**：保持同一主体、视角、尺度和环境，只改变被解释的变量。
- 核心是**机制**：优先桌面实物演示或低噪音界面，让物体状态、路径或界面反馈承担变化。

### 构图

- 优先俯视桌面、肩后视角、操作近景或清晰中景。
- 概念节点不超过四个。
- 正文图不承担整篇文章的情绪高潮，不设计成电影海报。
- 允许必要留白，但解释关系优先于标题安全区。
- 文字不是主要信息载体；模型可能生成不可靠文字时，改用位置、形状、材质和状态表达。

### 风格

- 清晰、克制、现实、低噪音。
- 生活与教育主题默认自然暖光；工业、分析和风险主题使用中性或低饱和光线。
- 视觉冲击与解释效率冲突时，选择解释效率。

## 从观点结构到现实场景

| 观点结构 | 优先现实场景 | 可观察动作 |
|---|---|---|
| 能力提升 | 工作台、课堂、创作桌 | 同一人物完成过去受限的任务 |
| 资源重新分配 | 项目桌、会议桌、生产台 | 手把筹码或任务卡移向关键位置 |
| 瓶颈 | 物流台、服务器机架、任务板 | 队列在一个狭窄节点等待 |
| 自动化 | 商店、办公室、家庭桌面 | 人完成判断，系统处理重复步骤 |
| 透明度 | 检查台、维修桌、审核现场 | 人打开、核对或追踪内部过程 |
| 信任与控制 | 驾驶舱、医疗台、控制室 | 人确认、接管或阻断错误路径 |
| 成本转移 | 预算桌、设备维护现场 | 重量、时间卡或资源移到另一节点 |
| 长期价值 | 家庭、办公室、维修台 | 用户继续使用有痕迹的旧设备 |
| 行业分化 | 同一生产线、共享入口 | 多个参与者沿不同路径分流 |
| 未确定趋势 | 原型现场、早期使用场景 | 人开始准备，路径仍未完成 |

## 分类、意图与情绪的组合顺序

1. **分类**决定现实对象和地点。
2. **作者意图**决定画面任务：呈现变化、解释机制、展示证据、复现步骤、比较体验或暗示趋势。
3. **情绪**只调整距离、动作力度、光线、对比和色彩。
4. **证据**限制画面可以表达多强，不允许情绪越过证据。

示例：`GPU + 分析 + 风险`不等于“红色发光显卡”。应先选择数据中心或任务分配桌面，用一个队列在有限计算槽位前等待，再以低饱和色和小面积琥珀提示瓶颈。

## 失败时如何重选

- **只有概念，没有动作**：把概念改写成人的操作，或物体自身可见的位置、数量、路径与状态变化。
- **出现两个焦点**：保留最能证明核心观点的一个，删除另一个。
- **需要大量文字解释**：改用真实物件的位置、数量、路径或状态变化。
- **像广告**：移除品牌中心、完美表面和夸张光效，加入真实使用动作。
- **像电影海报**：缩小事件规模，回到工作台、办公室、商店、教室或现场操作。
- **像抽象信息图**：用纸卡、筹码、容器、文件和手部操作重建关系。
- **情绪比事实更强**：降低对比和动作强度，恢复文章的限制条件。
